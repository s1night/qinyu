# 轻遇 - 产品需求文档

## Overview

- **Summary**: 轻遇是一款面向年轻人的轻社交平台，旨在解决年轻人不爱社交、不愿出门的问题，通过兴趣小组和轻出门广场功能，提供低压力、轻松自然的社交体验。
- **Purpose**: 创造一个让年轻人感到舒适的社交环境，鼓励他们通过兴趣匹配和轻松互动，逐步走出舒适区，建立真实的社交关系。
- **Target Users**: 18-35岁的年轻人，喜欢宅家、不爱社交但渴望建立真实联系的人群。

## Goals

- 建立轻松无压力的社交平台，降低社交门槛
- 通过兴趣匹配帮助用户找到志同道合的朋友
- 鼓励用户通过轻出门活动逐步走出舒适区
- 提供安全、隐私保护的社交体验

## Non-Goals (Out of Scope)

- 不提供复杂的社交游戏和任务系统
- 不做传统的陌生人随机匹配功能
- 不支持大型线下活动组织（超过10人的活动）
- 不涉及付费会员体系

## Background & Context

- 当前年轻人面临社交焦虑，传统社交平台压力大，匹配机制不够精准
- 年轻人更倾向于基于兴趣的社交，而非盲目扩大社交圈
- 需要平衡线上互动和线下活动，鼓励适度社交而非过度社交

## Functional Requirements

- **FR-1**: 用户管理 - 微信扫码登陆，个人主页展示，兴趣标签设置
- **FR-2**: 兴趣小组 - 创建/加入小组，发帖讨论，匿名互动
- **FR-3**: 轻出门广场 - 发布出门任务，距离显示，报名/拒绝机制
- **FR-4**: 隐私保护 - 匿名发帖/回复，个人信息控制

## Non-Functional Requirements

- **NFR-1**: 性能要求 - 页面加载时间<2秒，响应时间<500ms
- **NFR-2**: 安全要求 - 用户数据加密存储，隐私信息保护
- **NFR-3**: 用户体验 - 界面简洁易用，操作流程简单
- **NFR-4**: 可扩展性 - 支持未来功能扩展和用户规模增长

## Constraints

- **Technical**: React 19 + TypeScript + Next.js + PostgreSQL，支持Vercel部署，移动端优先设计
- **Business**: 初期聚焦核心功能，快速验证用户需求
- **Dependencies**: 微信开放平台API，地理位置服务，Vercel部署环境

## Assumptions

- 用户拥有智能手机和微信账号
- 用户愿意分享基本位置信息用于距离计算
- 用户希望在保护隐私的前提下进行社交互动

## Acceptance Criteria

### AC-1: 微信扫码登陆

- **Given**: 用户未登录状态
- **When**: 点击微信扫码登录按钮
- **Then**: 显示微信扫码界面，扫码后自动登录并跳转到首页
- **Verification**: `human-judgment`

### AC-2: 个人主页设置

- **Given**: 用户已登录
- **When**: 进入个人主页编辑界面
- **Then**: 可以编辑个人描述、兴趣标签，保存后立即生效
- **Verification**: `human-judgment`

### AC-3: 兴趣小组创建

- **Given**: 用户已登录
- **When**: 点击创建小组按钮，填写小组信息
- **Then**: 成功创建小组，用户成为管理员
- **Verification**: `programmatic`

### AC-4: 小组帖子发布

- **Given**: 用户已加入小组
- **When**: 发布帖子，选择是否匿名
- **Then**: 帖子成功发布，匿名模式下显示匿名标识
- **Verification**: `human-judgment`

### AC-5: 匿名回复功能

- **Given**: 用户在小组帖子详情页
- **When**: 点击回复，选择匿名回复
- **Then**: 回复成功，显示为匿名用户
- **Verification**: `human-judgment`

### AC-6: 轻出门任务发布

- **Given**: 用户已登录并允许位置权限
- **When**: 发布出门任务，设置人数限制（最多3人）
- **Then**: 任务成功发布，显示在广场页面
- **Verification**: `programmatic`

### AC-7: 距离显示功能

- **Given**: 用户查看出门任务列表
- **When**: 浏览任务卡片
- **Then**: 每个任务显示发起人距离自己的距离
- **Verification**: `human-judgment`

### AC-8: 任务报名与拒绝

- **Given**: 用户看到感兴趣的出门任务
- **When**: 点击报名按钮，发起人审核报名请求
- **Then**: 发起人可以接受或拒绝报名，报名者收到通知
- **Verification**: `programmatic`
